Description of the Github Workflow

  1. Commit operation sends the current changes on our project to our github
  2. Branch includes it is own changes before merge to the master branch and it helps to see the details that made in that current branch
  3. Pull request provides to check the changes with the other developers before it gets proof of the merge to the master branch.
  4. If we are working as a team we need to make sure to pull the changes from the branch we working on. It will make the project to be followed correctly and avoid the missing changes

Description of the Unix Shell

  1. ls command will list all files in the current directory
  2. pwd command will show the working directory and cd command will change the name of the directory
  3. mkdir command will create a new directory
